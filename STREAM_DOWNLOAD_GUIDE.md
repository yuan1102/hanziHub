## 概述

汉字Hub 现已支持**边下边播（Stream Download）**，用户无需等待视频完整下载，即可在下载过程中进行播放。

### 核心优势

✅ **无需等待** — 可立即开始播放
✅ **渐进加载** — 边下边播，提升体验
✅ **智能缓存** — 下载完成后保留本地缓存
✅ **自动降级** — 下载失败自动在线播放
✅ **网络友好** — 支持 HTTP Range 请求

## 缓存管理

### 文件生命周期

```
1. 开始下载
   └─ 创建临时文件：video_cache/绿.mp4.tmp

2. 边下边写
   └─ 每 100KB 保存一次到临时文件

3. 开始播放（当文件足够大时）
   └─ 播放器开始读取临时文件

4. 下载完成
   └─ 重命名：绿.mp4.tmp → 绿.mp4（最终缓存）

5. 后续播放
   └─ 直接读取缓存文件：video_cache/绿.mp4
```

### 临时文件清理

```dart
// 如果下载失败，删除不完整的临时文件
try {
  final tempFile = File('$cachePath.tmp');
  if (await tempFile.exists()) {
    await tempFile.delete();
  }
} catch (_) {}
```

## 最佳实践

### ✅ 推荐做法

1. **使用 HTTPS**
   - 所有视频 URL 使用 HTTPS
   - 确保传输安全

2. **合理的超时设置**
   - 5 分钟超时适合大多数场景
   - 可根据目标用户网络环境调整

3. **清晰的 UI 反馈**
   - 显示"正在下载"和"首次播放需要一点时间"
   - 帮助用户理解等待原因

4. **定期清理缓存**
   - 建议用户每月清理一次
   - 监控缓存大小

### ❌ 避免做法

- 不要设置过短的超时（如 10 秒）可能导致正常下载失败
- 不要频繁删除临时文件（可能破坏下载过程）
- 不要跳过网络检查（应该在网络可用时再下载）

## 故障排除

### ❌ 视频一直显示"正在下载"

**可能原因**：
- 网络连接差
- 服务器不可达
- 超时时间设置过短

**解决方案**：
1. 检查网络连接
2. 确保 URL 正确且服务器可访问
3. 增加超时时间

### ❌ 播放时出现卡顿

**可能原因**：
- 网络速度太慢
- 文件保存频率过高

**解决方案**：
1. 检查网络速度
2. 减少保存频率（如从每 100KB 改为每 500KB）

### ❌ 临时文件没有删除

**可能原因**：
- 下载中断
- 应用强制关闭

**解决方案**：
1. 手动清理：设置 → 缓存管理 → 清空缓存
2. 或删除 app_cache/video_cache/*.tmp 文件

## 用户指南

### 什么是边下边播？

边下边播是指视频无需完整下载，用户可以在下载过程中进行播放：
- **第一次播放**：需要等待 1-3 秒，视频足够多后开始播放
- **后续播放**：直接从缓存读取，0.5 秒即可开始

### 为什么首次播放需要等待？

- 需要一点时间下载足够的视频数据
- 这样可以避免网络卡顿导致的播放中断
- 通常只需等待 1-3 秒

### 如何加快播放速度？

1. **使用 WiFi**（速度更快）
2. **接近 WiFi 路由器**（信号更强）
3. **关闭其他网络应用**（避免占用带宽）
4. **在中国使用 CDN**（速度会更快）

### 缓存的视频可以离线观看吗？

是的！缓存后的视频可以在没有网络时观看。

## 常见问题

### Q: 边下边播会损坏视频吗？

A: 不会。临时文件正在下载，完成后才重命名为最终文件。即使中断，下次会重新下载。

### Q: 可以暂停下载吗？

A: 当前不支持。后续版本可以实现暂停和继续功能。

### Q: 多个视频同时下载可以吗？

A: 可以。但会占用更多网络带宽，可能导致播放卡顿。

### Q: 如何禁用边下边播？

A: 修改代码，在 `_initializePlayer` 中改为等待完整下载。

### Q: 边下边播适合所有网络吗？

A: 最适合 WiFi（10+ Mbps）。如果网络很慢（< 1 Mbps），播放可能卡顿。
