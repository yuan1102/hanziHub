## 技术实现

### 智能播放策略

```dart
// 检查缓存存在状态
if (await videoFile.exists()) {
  // 方案 A：使用缓存（后续播放）
  media = Media(cachedPath);
} else {
  // 方案 B：直接播放在线（首次播放）
  media = Media(videoUrl);

  // 同时静默下载缓存（不显示进度，不阻塞 UI）
  _silentDownloadVideo(videoUrl);
}
```

### 静默后台下载

```dart
/// 静默后台下载（用户无感）
Future<void> _silentDownloadVideo(String videoUrl) async {
  try {
    final cachedPath = await VideoCacheManager.getOrDownloadVideo(videoUrl);
    if (cachedPath != null) {
      debugPrint('✓ 视频已缓存');
    }
  } catch (e) {
    // 失败时静默处理，不影响在线播放
    debugPrint('后台下载失败（忽略）: $e');
  }
}
```

## 常见问题

### Q: 首次为什么还要播放在线视频？

A: 因为这样可以即时响应，用户体验最好。后台同时下载缓存，后续直接用缓存。

### Q: 缓存下载失败会怎样？

A: 不影响，用户已在线播放了。后续播放时仍会在线播放。可再试一次。

### Q: 如何加快缓存下载？

A:
1. 使用 WiFi（速度更快）
2. 靠近路由器（信号更强）
3. 关闭其他应用（避免占用带宽）

### Q: 缓存后可以离线观看吗？

A: 是的，缓存视频可以在没有网络时观看。

### Q: 如何禁用缓存？

A: 可以在设置中的缓存管理中随时清理缓存。

### Q: 为什么有时还是很卡？

A: 可能是网络速度慢。使用高速网络会更流畅。

## 最佳实践

### ✅ 推荐

1. **保持网络连接** — 首次播放需要网络
2. **定期清理缓存** — 检查存储空间
3. **使用 WiFi** — 下载速度更快
4. **耐心等待** — 低速网络需要时间

### ❌ 避免

- 频繁删除缓存（没必要，自动管理更好）
- 在弱网环境下期望快速缓存
- 一次打开太多视频（会卡）
